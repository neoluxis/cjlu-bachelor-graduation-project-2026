\section{研究历史回顾}
焊缝缺陷的自动化检测研究早期主要依赖数字图像处理与人工特征工程，通过滤波去噪、形态学处理、边缘检测与纹理特征等方法实现缺陷增强与识别。这类方法在成像质量较高、背景相对简单的场景中具备一定有效性，但面对工业射线图像常见的低对比度、噪声干扰与缺陷形态多样性时，鲁棒性与泛化能力不足，细小缺陷（如气孔）与弱对比缺陷（如裂纹）容易漏检 \cite{Valavanis2010_MulticlassDefectDetection, Kumar2014_TextureFeatureWeldFlaws, Hu2020_DeepLearningWeldDefect}。

随后，基于浅层学习器（如 SVM、传统神经网络）的方案在一定程度上提升了分类效果，但仍高度依赖手工特征设计与数据分布假设，难以适配工业现场多工况变化。近年来，深度学习以端到端特征学习能力成为主流路线：从早期 CNN 分类网络，到两阶段/一阶段目标检测模型，再到语义分割模型与端到端系统集成，研究焦点逐步从“是否缺陷”走向“缺陷在哪里、形状如何、面积多大”的精细化分析 \cite{He2015_DeepResidualLearning, Girshick2015_FastRCNN, Ren2017_FasterRCNN, Ronneberger2015_UNet}。

\section{研究现状对比}
现有研究主要可从“图像预处理”和“缺陷识别模型”两条主线展开。
\begin{enumerate}[leftmargin=*]
  \item \textbf{预处理与增强：} 传统方法（高斯/中值滤波、直方图均衡化等）实现简单、可解释性强，但对不同质量图像往往需要人工调参，适应性不足。面向低质量射线图像的自适应预处理与增强正成为热点 \cite{Hou2020_ReviewWeldDefectDetection, Hu2020_DeepLearningWeldDefect, Guo2021_ContrastEnhancementGAN}。
  \item \textbf{检测/分割模型：} 目标检测模型可同时输出缺陷类别与位置，适用于工程场景快速定位，但对缺陷边界与形态刻画有限。语义分割能够给出像素级缺陷区域，便于后续定量评估与严重度分析。针对工业小样本与多尺度缺陷，研究者在网络结构、特征融合与正负样本不平衡等方面持续改进，例如改进 Faster R-CNN 以适配小样本与小目标 \cite{Ajmi2023_FasterRCNNWeldDefects}，以及通过级联检测、特征金字塔与特征擦除等机制提升多尺度缺陷识别能力 \cite{Dai2025_CCBFERCNN}。
  \item \textbf{迁移学习与预训练：} 工业射线图像与自然图像存在显著领域差异，单纯依赖 ImageNet 预训练可能受限。利用医学射线图像等“成像机理相近”的数据进行预训练并迁移到工业缺陷任务，是缩小领域差异、提升泛化性能的有效思路 \cite{Yu2025_IndustrialDefectRecognition, Irvin2019_CheXpertDataset, Li2010_ImageNet}。
\end{enumerate}


\section{发展趋势与本课题方向}
总体趋势是从“检测定位”向“精细分割与系统集成”演进：一方面，工业质量控制需要对缺陷区域进行更精确的边界提取与量化分析；另一方面，部署端对实时性、稳定性与可用性交付提出要求，促使研究从单一模型性能指标扩展到包含数据预处理、模型推理、结果可视化与人机交互的端到端流程优化 \cite{Zuo2025_EndToEndWeldingDefect, Yu2025_IndustrialDefectRecognition}。

本课题面向螺旋焊接管道 X 射线图像，围绕“缺陷分割+系统实现”开展研究：在数据集（SWRD 用于训练、RIAWELC 用于跨数据集测试）条件下，先对多种预处理方法进行对比选择，再构建基于 SegFormer 的缺陷语义分割模型，实现不少于五类缺陷的像素级分割；同时与多种主流语义分割模型进行对比实验，验证模型的鲁棒性与泛化能力，并基于 Qt 图形界面实现检测流程与结果可视化。

\section{选题的背景与意义}
\subsection{研究背景}
焊缝质量直接影响压力容器、长输管道等关键装备的结构安全与寿命，焊接过程受工艺波动、设备误差与材料差异等因素影响，易产生气孔、裂纹、夹渣、未熔合/未焊透等内部缺陷 \cite{Hu2020_DeepLearningWeldDefect, Dai2025_CCBFERCNN}。X 射线无损检测具有成像直观、定位准确、不破坏工件等优势，是焊缝内部缺陷检测的重要手段 \cite{Mery2015_GDXrayDataset, Yu2025_IndustrialDefectRecognition}。

\subsection{研究意义}
传统射线评片高度依赖人工经验，效率低且易受主观差异与视觉疲劳影响，在低亮度、低对比度或雾化等低质量图像场景中漏检误判风险更高 \cite{Hou2020_ReviewWeldDefectDetection, Ajmi2023_FasterRCNNWeldDefects}。引入深度学习与计算机视觉方法，可在大规模数据上学习更具判别力的特征表示，提升检测的自动化程度与一致性，并为缺陷的定量评估与追溯分析提供基础 \cite{He2015_DeepResidualLearning, Yu2025_IndustrialDefectRecognition}。因此，开展基于 SegFormer 的射线图像缺陷分割及其工程化检测系统设计，具有明确的工程应用价值与研究意义。

\section{国内外发展现状}
国内外相关研究已在数据集建设、预处理增强、检测/分割模型与系统集成等方面取得进展。预处理方面，滤波去噪、直方图均衡化与对比度增强等方法广泛应用，但固定参数策略难以覆盖不同成像质量与工况变化；面向低质量射线图像的自适应增强与联合优化逐步受到关注 \cite{Hou2020_ReviewWeldDefectDetection, Guo2021_ContrastEnhancementGAN}。模型方面，从 CNN 分类到目标检测与语义分割框架不断演进：两阶段检测（Faster R-CNN 等）在小样本条件下仍具竞争力 \cite{Ajmi2023_FasterRCNNWeldDefects}，多尺度特征融合与正则化策略可提升复杂缺陷识别能力 \cite{Dai2025_CCBFERCNN}；语义分割能够给出像素级缺陷区域，满足缺陷形态量化分析需求 \cite{Ronneberger2015_UNet, Yu2025_IndustrialDefectRecognition}。此外，跨领域预训练与迁移学习用于缓解工业样本稀缺与领域差异问题，医学射线图像预训练提升工业缺陷分割性能的思路具有代表性 \cite{Yu2025_IndustrialDefectRecognition, Irvin2019_CheXpertDataset}。

综合来看，仍存在若干待解决问题：低质量图像下的稳健分割、跨数据集泛化能力提升、小样本条件下的可靠训练策略，以及将模型能力以可用的交互式系统形式落地等。

\section{研究的基本内容与拟解决的主要问题}
\begin{enumerate}[leftmargin=*]
  \item \textbf{数据与预处理：} 采用指定螺旋焊接管道 X 射线图像数据集，以 SWRD 数据集用于训练，在 RIAWELC 数据集上进行跨数据集测试；对不少于三种预处理方法（如高斯滤波去噪、CLAHE 直方图均衡化等）进行对比分析，选取最适合数据分布的预处理方案。
  \item \textbf{缺陷分割模型：} 构建基于 SegFormer 的缺陷语义分割模型，实现不少于五类缺陷（如气孔、夹渣、裂纹、咬边、未熔合/未焊透等）的像素级分割；以 IoU、像素准确率（PA）与 mIoU 为核心指标，目标达到 IoU>0.7、PA>0.75、mIoU>0.8。
  \item \textbf{对比实验与泛化验证：} 在相同数据与评估协议下，与不少于三种主流语义分割深度学习模型（如 DeepLab v3、Mask R-CNN、EfficientNet v2 等）进行对比实验，分析不同模型在多尺度缺陷、复杂背景与低质量图像条件下的性能差异；通过跨数据集测试评估鲁棒性与泛化能力。
  \item \textbf{系统设计与实现：} 基于 Qt 图形界面实现射线图像导入、缺陷检测触发、分割结果可视化与结果导出等功能，形成可运行的缺陷识别与检测系统原型。
  \item \textbf{拟解决的关键问题：} （1）工业射线图像低对比度与噪声干扰下缺陷边界难以分离；（2）小样本与类别不平衡导致训练不稳定、易过拟合；（3）跨数据集存在领域差异，模型泛化能力不足；（4）模型推理与界面交互的工程集成与稳定性保障。
\end{enumerate}

\section{技术路线、研究方案与可行性分析}
\begin{enumerate}[leftmargin=*]
  \item \textbf{系统整体设计：} 按“数据管理与预处理模块-模型训练与推理模块-结果评估与可视化模块-交互界面模块”进行分层设计；训练侧完成数据划分、预处理与模型训练，部署侧提供单张/批量推理与结果导出。
  \item \textbf{研究方案（算法流程）：} （1）数据清洗与标注核验；（2）预处理方法对比与参数确定；（3）SegFormer 模型训练（含数据增强、损失函数与超参设定）；（4）在测试集与跨数据集上评估（IoU、PA、mIoU 等）；（5）与多种基线模型进行对比实验；（6）导出推理模型并封装推理接口，接入 Qt 界面完成可视化与交互流程。
  \item \textbf{实施方案（工程实现）：} 以 Python 深度学习框架完成模型训练与推理（训练脚本、评估脚本、模型导出与版本管理）；界面侧采用 Qt（如 PyQt/PySide 或 C++ Qt）实现“导入-检测-展示-导出”的闭环交互，并提供异常提示与日志记录以提升可用性。
  \item \textbf{可行性分析：} 数据与指标来源明确（SWRD/RIAWELC 与任务书指标约束）；方法上以成熟的语义分割范式为基础，并通过预处理对比、对比实验与跨数据集测试降低方案风险；工程上 Qt 具备成熟的 GUI 生态，模型推理可通过离线批处理与单张交互两种模式满足效率需求。若本地算力不足，可采用云端 GPU 进行训练，部署阶段以较低资源完成推理。
\end{enumerate}
